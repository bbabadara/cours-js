Leçon 2/10

Les Variables

Les variables sont des conteneurs qui permettent de stocker des données.

Différence entre let, const et var

  • let : portée de bloc, peut être modifié
  • const : constante (ne peut pas être modifié)
  • var : portée fonctionnelle (ancienne méthode)

Exemples

Déclarer avec let

let nom = "Alice";
let age = 25;
console.log(nom);    // Alice

Déclarer avec const

const PI = 3.14159;
const MESSAGE = "Bienvenue";
console.log(PI);

Leçon 5/10

Boucles

Les boucles permettent de répéter du code plusieurs fois.

for

Idéale quand on connaît le nombre d'itérations.

for (let i = 0; i < 3; i++) {
    console.log("Itération " + i);
}

while

Répète tant que la condition est vraie.

let i = 0;
while (i < 3) {
    console.log("Compteur: " + i);
    i++;
}

for...of

Itère sur les éléments d'un tableau.

let fruits = ["pomme", "banane"];

for (let fruit of fruits) {
    console.log(fruit);
}

Leçon 11/14

Formulaires

Les formulaires permettent de collecter des données utilisateur.

Événements de formulaire

  • submit - Soumission du formulaire
  • input - Modification d'un champ
  • change - Changement de valeur
  • focus - Focus sur un champ
  • blur - Perte du focus

Récupérer les valeurs

// Via FormData
const form = document.getElementById("monForm");
form.addEventListener("submit", function(e) {
    e.preventDefault();
    const data = new FormData(form);
    console.log(data.get("nom"));
    console.log(data.get("email"));
});

// Directement
const email = document.getElementById("email").value;
console.log(email);

Validation

// Vérifier si un champ est rempli
if (nom.value === "") {
    alert("Le nom est requis");
}

// Vérifier le format email
const email = "test@email.com";
const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
console.log(emailRegex.test(email));

Exemple complet

const form = document.querySelector("form");
form.addEventListener("submit", function(e) {
    e.preventDefault();
    const nom = document.getElementById("nom").value;
    const email = document.getElementById("email").value;
    console.log("Soumis:", nom, email);
});
Leçon 12/14

localStorage

localStorage permet de stocker des données persistantes dans le navigateur.

Caractéristiques

  • Données persistantes (jamais expirées)
  • Stockage maximal ~5MB
  • Stocke uniquement des chaînes
  • Accessible depuis le même domaine

Méthodes

setItem() - Sauvegarder

localStorage.setItem("nom", "Alice");
localStorage.setItem("age", "25");

getItem() - Récupérer

const nom = localStorage.getItem("nom");
console.log(nom);

removeItem() - Supprimer

localStorage.removeItem("nom");
localStorage.clear(); // Tout effacer

Stocker des objets

JSON.stringify() et JSON.parse()

const utilisateur = {
    nom: "Alice",
    age: 25,
    ville: "Paris"
};

// Sauvegarder (convertir en JSON)
localStorage.setItem("utilisateur", JSON.stringify(utilisateur));

// Récupérer (convertir en objet)
const data = JSON.parse(localStorage.getItem("utilisateur"));
console.log(data.nom);

Leçon 13/14

sessionStorage

sessionStorage fonctionne comme localStorage mais les données sont effacées à la fermeture du navigateur.

Différence avec localStorage

  • Données effacées à la fermeture de l'onglet
  • Méthodes identiques (setItem, getItem, removeItem)
  • Utile pour des données temporaires

Utilisation

// Même syntaxe que localStorage
sessionStorage.setItem("token", "abc123");
sessionStorage.setItem("panier", JSON.stringify([
    { produit: "Livre", prix: 15 },
    { produit: "Cahier", prix: 5 }
]));

const token = sessionStorage.getItem("token");
const panier = JSON.parse(sessionStorage.getItem("panier"));

console.log("Token:", token);
console.log("Panier:", panier);

Quand utiliser sessionStorage ?

  • Panier d'achat temporaire
  • Token de session
  • Données de formulaire non soumises
  • State temporaire d'une application
Leçon 14/14

Programmation Orientée Objet

La POO permet de structurer le code autour d'objets.

Concepts clés

  • Classe - Plan de construction
  • Objet - Instance d'une classe
  • Héritage - Réutiliser le code
  • Encapsulation - Cacher les données

Créer une classe

class Personne {
    constructor(nom, age) {
        this.nom = nom;
        this.age = age;
    }
    
    saluer() {
        return "Bonjour, je suis " + this.nom;
    }
}

// Créer un objet
const alice = new Personne("Alice", 25);
console.log(alice.nom);
console.log(alice.saluer());

Héritage

Créer une classe basée sur une autre.

class Animal {
    constructor(nom) {
        this.nom = nom;
    }
    
    parler() {
        return "...";
    }
}

class Chien extends Animal {
    constructor(nom, race) {
        super(nom); // Appeler le constructeur parent
        this.race = race;
    }
    
    parler() {
        return "Woof !";
    }
}

const medor = new Chien("Médor", "Labrador");
console.log(medor.nom);
console.log(medor.parler());

Getters et Setters

class Rectangle {
    constructor(largeur, hauteur) {
        this.largeur = largeur;
        this.hauteur = hauteur;
    }
    
    // Getter
    get surface() {
        return this.largeur * this.hauteur;
    }
    
    // Setter
    set dimensions({largeur, hauteur}) {
        this.largeur = largeur;
        this.hauteur = hauteur;
    }
}

const rect = new Rectangle(10, 5);
console.log("Surface:", rect.surface);

rect.dimensions = {largeur: 20, hauteur: 10};
console.log("Nouvelle surface:", rect.surface);

Leçon 15/16

Fetch API

Fetch API permet de faire des requêtes HTTP pour récupérer ou envoyer des données.

Pourquoi utiliser Fetch ?

  • Récupérer des données d'une API
  • Envoyer des données au serveur
  • Plus moderne que XMLHttpRequest
  • Basé sur les Promises

fetch() - Récupérer des données

// Requête GET simple
fetch('https://api.example.com/data')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Erreur:', error));

fetch() - Envoyer des données

Avec la méthode POST.

fetch('https://api.example.com/users', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        nom: 'Alice',
        email: 'alice@email.com'
    })
})
    .then(response => response.json())
    .then(data => console.log(data));

Options de fetch

method GET, POST, PUT, DELETE
headers En-têtes HTTP
body Données à envoyer
mode cors, no-cors

Leçon 16/16

Async / Await

Async/Await est une syntaxe plus élégante pour gérer les opérations asynchrones.

Avantages

  • Plus lisible que les .then()
  • Gestion d'erreurs simplifiée avec try/catch
  • Code qui ressemble à du code synchrone

async function

async function recupererDonnees() {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
}

// Appeler la fonction
recupererDonnees().then(data => console.log(data));

try / catch

Pour gérer les erreurs.

async function recupererDonnees() {
    try {
        const response = await fetch('https://api.example.com/data');
        if (!response.ok) {
            throw new Error('Erreur HTTP');
        }
        const data = await response.json();
        return data;
    } catch (error) {
        console.error('Erreur:', error);
    }
}

await avec Promise.all()

Pour exécuter plusieurs requêtes en parallèle.

async function chargerTout() {
    const [users, posts] = await Promise.all([
        fetch('/api/users').then(r => r.json()),
        fetch('/api/posts').then(r => r.json())
    ]);
    
    console.log('Users:', users);
    console.log('Posts:', posts);
}
